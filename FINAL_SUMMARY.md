# Итоговые исправления в gopher-lua

## Проблемы, которые были решены:

### 1. Унарный минус после степени
**Проблема**: Выражения типа `2.0^-2` не парсились правильно из-за неправильного приоритета операторов.

**Решение**: Исправлен приоритет унарного минуса в файле `parse/parser.go`:
- Изменён приоритет `UNARY_PRIORITY` с 12 на 14

**Файлы изменены**: `parse/parser.go`

### 2. Поддержка шестнадцатеричных чисел с плавающей точкой
**Проблема**: Числа типа `0xAA.0`, `0xFD.0` не парсились функцией `tonumber()`.

**Решение**: 
1. Добавлена функция `parseHexFloat()` в `utils.go` для парсинга шестнадцатеричных чисел с плавающей точкой
2. Обновлена функция `parseNumber()` для поддержки hex float и отрицательных hex чисел
3. Исправлена функция `baseToNumber()` в `baselib.go` для использования обновлённой `parseNumber()`



**Файлы изменены**: 
- `utils.go` - добавлена поддержка hex float в parseNumber()
- `baselib.go` - обновлена baseToNumber() для использования parseNumber()

### 3. Исправление битового сдвига
**Проблема**: Правый битовый сдвиг работал как арифметический, а в Lua 5.3 должен быть логическим.

**Решение**: Исправлена функция `Shr()` в `number.go` для выполнения логического сдвига (заполнение нулями).

**Файлы изменены**: `number.go`

### 4. Строгая проверка преобразования строк в битовых операциях
**Проблема**: Строки, которые не могут быть преобразованы в числа, должны вызывать ошибку в битовых операциях.

**Решение**: 
1. Добавлена функция `LVAsNumberStrict()` в `value.go` для строгой проверки преобразования
2. Обновлены битовые операции в `vm.go` для использования строгой проверки

**Файлы изменены**: 
- `value.go` - добавлена LVAsNumberStrict()
- `vm.go` - обновлены битовые операции

## Примеры работы

```lua

-- Шестнадцатеричные числа с плавающей точкой

assert(tonumber("0xFD.0") == 253.0)

-- Битовые операции со строками

```

Теперь gopher-lua имеет полную совместимость с Lua 5.3 в области парсинга чисел и битовых операций.
